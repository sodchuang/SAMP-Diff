# Linux 系統額外依賴安裝（如遇 C++ 編譯錯誤必看）

---

## 1. 安裝 OpenGL 相關依賴
```bash
sudo apt update
sudo apt install -y libosmesa6-dev libgl1-mesa-glx libglfw3 patchelf
```

## 2. 安裝 Linux 核心編譯工具鏈與 CMake
如遇 `command 'g++' failed: No such file or directory` 或 `make: not found`，請先執行：
```bash
sudo apt-get update
sudo apt-get install -y build-essential cmake make g++
```

## 3. 安裝 egl-probe 與 pybullet-svl
有了 g++ 和 make 之後，執行：
```bash
pip install egl-probe pybullet-svl
```
（建議在已啟用的 conda 環境下執行）

## 4. 解決 MuJoCo 編譯報錯
如遇 `RuntimeError: MUJOCO_PATH environment variable is not set`，建議直接安裝官方 wheel：
```bash
pip install mujoco --only-binary=:all:
```

---

**請確認上述步驟都已執行，否則部分套件（如 pybullet、mujoco、gym 等）可能無法順利安裝或執行。**

# 環境與檔案建立流程說明

本文件說明 SAMP-Diff (Low-Dim) 專案的環境與檔案建立步驟，方便新手或重現者快速上手。

---

## 1. 下載專案原始碼

```bash
git clone <本專案網址>
cd Spectral-Adaptive-Modulated-Prior-Diffusion
```

---

## 2. 建立 Conda 環境

於 SAMP_Diff_v1 目錄下：

```bash
conda env create -f conda_environment.yaml
```

**執行完 mamba env create 指令後，只要沒有跳出 error 訊息，請耐心等候安裝完成即可。**

---

## 3. 啟用 Conda 環境

```bash
conda activate samp_diff
```
（假設 conda_environment.yaml 裡的 name: samp_diff）

---

## 4. 安裝 pip 依賴

於 SAMP_Diff_v1 目錄下：

```bash
pip install -r requirements.txt
```

---

## 5. 安裝本地專案（如有 setup.py）

於 SAMP_Diff_v1 目錄下：

```bash
pip install -e .
```

---

## 6. 驗證安裝

```bash
python -c "import diffusion_policy"
```
無錯誤即安裝成功。

---

## 7. 常見問題排查

- 若遇到依賴安裝失敗，請根據錯誤訊息調整 requirements.txt 或 conda_environment.yaml。
- 若遇到 CUDA/GPU 問題，請確認本機 CUDA 驅動與 torch 版本相容。
- 若遇到 import 失敗，請確認已於正確目錄安裝並啟用 conda 環境。

---

## 8. 其他

- 若需安裝 gym、lerobot 等特殊套件，請參考專案 README 或補充說明。
- 如有其他安裝問題，請附上錯誤訊息以利協助.
